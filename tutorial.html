2. Meshing tutorialÂ¶

Roughly said, an FE software understands geometry as mesh elements and the connection between these elements. There are many tools that can convert a CAD model into an FE mesh. Each FE tool has its own mesh format. In this workshop, we’ll be working with LS-DYNA. LS-PrePost is a freeware that can convert such a CAD model into mesh elements understandable by LS-DYNA. Since using LS-PrePost can be quite time-consuming for the purposed of our tutorial, we’ll be using the scripting language Python to help us in the task of meshing our geometry.

That said, our geometry will be simple [solid] cuboids and [shell] rectangles as shown below:

The python code that generates the above two geometry is called as follows:

createMesh --x <X> --y <Y> --z <Z> --nx <NX> --ny <NY> --nz <NZ> --geo <GEOMETRY_FILE.k> --[shell | solid]

The variables inside <> must be replaced with your values. These are the meanings of the arguments that you find in the command line:

Argument Description
--x Length of the geometry in the X-axis
--y Width of the geometry in the Y-axis
--z Height of the geometry in the Z-axis
--nx Number of elements in the X-axis
--ny Number of elements in the Y-axis
--nz Number of elements in the Z-axis
--geo Name of the geometry file. The file suffix (file extension) should be ‘.k’
--shell Creates a surface mesh
--solid Creates a solid mesh

There are some more arguments available in this python script. Check them out by typing:

createMesh --help
